- whitespace
- typography
- animations
- gradients
- clean layouts

## Technical Standards

Whenever modifying code:

- preserve functionality
- avoid unnecessary dependencies
- write clean HTML/CSS/JS
- keep components reusable
- minimize JavaScript
- optimize loading performance
- improve Lighthouse score whenever possible

## Accessibility

Always ensure

- semantic HTML
- keyboard navigation
- ARIA labels where appropriate
- sufficient color contrast
- responsive layouts
- readable typography

## GitHub Pages Compatibility

Never introduce technologies that GitHub Pages cannot host directly unless I specifically ask.

Prefer:

- HTML
- CSS
- Vanilla JS

If frameworks are introduced, explain why.
